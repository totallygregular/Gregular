Build a polished static website for GitHub Pages called “Character Vault”.

Goal:
Create a grid of interactive, glass-like character cards for game characters. Each card should show:
- A portrait image
- The character name centered over the image
- A subtle glass / frosted acrylic appearance
- A hover effect where the card tilts toward the cursor
- A moving reflective glare that follows the cursor like a light source
- A click effect where the card animates forward toward the user, flips over in 3D, and reveals a detailed back side

Tech stack and constraints:
- Use only HTML, CSS, and vanilla JavaScript
- No frameworks
- No build step
- Must run as a static GitHub Pages site
- Use semantic HTML and keep code clean and modular
- Organize into:
  - index.html
  - styles.css
  - script.js
  - assets/ for images
- Use relative paths only
- Make the first version desktop-first but responsive down to mobile

Design direction:
- Dark fantasy / premium ARPG feel
- Elegant, restrained, not gaudy
- Glass card effect should feel like smoked glass, not bright neon
- Background should be dark and atmospheric, but the card remains the main focus
- Typography should feel game-adjacent and premium, but still readable
- Avoid overly saturated gradients, cheesy glow spam, and generic “AI SaaS” design

Card behavior requirements:
1. Each card must have a front face and a back face
2. The front face contains:
   - portrait image
   - centered character name
   - subtle glass overlay
   - optional class/subclass label
3. The back face contains:
   - character name
   - game/class/build
   - short lore or notes
   - stat chips or tags
4. On mouse move over a card:
   - rotate slightly on X and Y axis based on cursor position
   - keep movement subtle and smooth
   - move a reflective highlight/glare across the card based on cursor coordinates
   - add a slight brightness shift to simulate the cursor as a light source
5. On mouse leave:
   - animate card smoothly back to neutral
   - fade glare back to a soft idle state
6. On click:
   - selected card scales slightly larger
   - card moves forward in 3D space
   - card flips to reveal the back face
   - non-selected cards should dim slightly
7. Clicking again or pressing a close button returns card to normal
8. Keyboard accessibility:
   - cards must be focusable
   - Enter or Space should trigger open/flip
   - Escape should close an expanded card

Technical implementation requirements:
- Use CSS perspective on the card container
- Use transform-style: preserve-3d on the inner flipping element
- Use backface-visibility: hidden on front and back faces
- Use translate / rotate / scale transforms for hover and flip behavior
- Use requestAnimationFrame or efficient event handling for mousemove animation
- Use CSS variables for cursor position, tilt amount, glare position, and glow intensity
- Use a pseudo-element or overlay layer for the moving reflective sheen
- Use semi-transparent backgrounds, soft borders, subtle inner highlights, and backdrop blur for the glass effect
- Keep animations smooth and tasteful with easing
- Respect prefers-reduced-motion

Accessibility and UX:
- Include visible focus styles
- Ensure text contrast remains readable over the portrait image
- Add a dark image overlay behind centered text if needed
- On touch devices, disable live tilt tracking and replace it with a lighter tap interaction
- Do not rely on hover for core functionality
- Ensure the layout still works without JavaScript, even if effects are reduced

Performance requirements:
- Keep JavaScript lightweight
- Avoid heavy particle systems or WebGL
- Use GPU-friendly transforms and opacity changes
- Lazy-load card images if practical
- Do not create laggy or overdone effects

Content requirements:
- Create 6 sample character cards using placeholder fantasy ARPG content
- Include one sample based on a “POE2 Blood Mage” style archetype, but keep the content original and not dependent on copyrighted game text
- Use placeholder images from local assets with clear filenames
- Make it easy for me to replace card data later

Deliverables:
- A working static site with a responsive card grid
- Clean HTML/CSS/JS files
- Comments only where helpful
- A short README section at the top of the response explaining:
  - file structure
  - how to replace images
  - how to add new cards
  - how to deploy to GitHub Pages

Visual detail targets:
- Cards should feel premium, tactile, and slightly cinematic
- Hover tilt should be subtle, around a few degrees only
- Glare should look like a soft reflective streak, not a hard white bar
- Flip animation should feel physical and deliberate
- The expanded/open state should feel like the chosen card is being presented to the player

Important:
- Do not use a generic glassmorphism template
- Do not use random floating blobs or neon purple gradients
- Make it feel like a game codex / character archive, not a startup landing page
- Prioritize polish over extra features

Quality bar:
Make the interaction feel like a premium trading card or character codex UI. The card should not wobble wildly. Favor subtlety, smooth easing, and restrained lighting. I care more about “feels expensive” than “shows off lots of effects”.

Art brief:
Imagine a dark fantasy codex interface with the polish of a premium game launcher. The cards look like tinted enchanted glass over painted portraits. The motion should feel physical and responsive, like the cursor is a lantern moving across lacquered glass. The flip/open animation should feel intentional, like presenting a rare card for inspection.
